# Структура базы данных

## 📁 Таблица `Users` (Пользователи)

Хранит информацию о пользователях приложения.

| Поле | Тип | Ограничения | Описание |
|------|-----|-------------|----------|
| `id` | INTEGER | PRIMARY KEY AUTOINCREMENT | Уникальный идентификатор пользователя |
| `name` | VARCHAR(64) | NOT NULL | Имя пользователя |
| `email` | VARCHAR(64) | UNIQUE NOT NULL | Электронная почта (уникальная) |
| `password_hash` | VARCHAR(255) | NOT NULL | Хеш пароля |
| `currency` | VARCHAR(3) | NOT NULL DEFAULT 'RUB' | Валюта пользователя по умолчанию (RUB, USD, EUR) |
| `created_at` | DATETIME | DEFAULT CURRENT_TIMESTAMP | Дата и время регистрации |

**Связи:** Один пользователь может иметь много категорий, счетов и транзакций.

---

## 📁 Таблица `Categories` (Категории)

Хранит категории доходов и расходов для каждого пользователя.

| Поле | Тип | Ограничения | Описание |
|------|-----|-------------|----------|
| `id` | INTEGER | PRIMARY KEY AUTOINCREMENT | Уникальный идентификатор категории |
| `user_id` | INTEGER | NOT NULL REFERENCES Users(id) | ID пользователя-владельца |
| `name` | VARCHAR(64) | NOT NULL | Название категории (например: "Продукты", "Зарплата") |

**Связи:** 
- `user_id` → `Users.id` (много категорий принадлежит одному пользователю)
- Одна категория может быть во многих транзакциях

---

## 📁 Таблица `Account` (Счета)

Хранит финансовые счета пользователей (наличные, банковские карты и т.д.).

| Поле | Тип | Ограничения | Описание |
|------|-----|-------------|----------|
| `id` | INTEGER | PRIMARY KEY AUTOINCREMENT | Уникальный идентификатор счета |
| `user_id` | INTEGER | NOT NULL REFERENCES Users(id) | ID пользователя-владельца |
| `name` | VARCHAR(64) | NOT NULL | Название счета (например: "Наличные", "Карта Сбера") |
| `spend_method` | VARCHAR(10) | NOT NULL DEFAULT 'CARD' | Способ траты: 'CARD' (карта) или 'CASH' (наличные) |
| `balance` | DECIMAL(10,2) | DEFAULT 0.00 | Текущий баланс на счете |

**Связи:**
- `user_id` → `Users.id` (много счетов у одного пользователя)
- Один счет может иметь много транзакций

---

## 📁 Таблица `Transaction` (Транзакции)

Хранит все финансовые операции (доходы и расходы).

| Поле | Тип | Ограничения | Описание |
|------|-----|-------------|----------|
| `id` | INTEGER | PRIMARY KEY AUTOINCREMENT | Уникальный идентификатор транзакции |
| `user_id` | INTEGER | NOT NULL REFERENCES Users(id) | ID пользователя |
| `account_id` | INTEGER | NOT NULL REFERENCES Account(id) | ID счета, с которого проведена операция |
| `category_id` | INTEGER | NOT NULL REFERENCES Categories(id) | ID категории операции |
| `amount` | DECIMAL(10,2) | NOT NULL | Сумма операции (положительная - доход, отрицательная - расход) |
| `description` | VARCHAR(256) | NULL | Описание/комментарий к операции |
| `transaction_date` | DATETIME | DEFAULT CURRENT_TIMESTAMP | Дата и время операции |

**Связи:**
- `user_id` → `Users.id` (транзакции принадлежат пользователю)
- `account_id` → `Account.id` (транзакция привязана к счету)
- `category_id` → `Categories.id` (транзакция привязана к категории)

---

## 🔗 Схема связей между таблицами
Users
├── Categories (один ко многим) - у пользователя много категорий
├── Account (один ко многим) - у пользователя много счетов
└── Transaction (один ко многим) - у пользователя много транзакций

Account → Transaction (один ко многим) - на одном счете много операций

Categories → Transaction (один ко многим) - в одной категории много операций
